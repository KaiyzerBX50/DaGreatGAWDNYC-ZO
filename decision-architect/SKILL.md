# Decision Architect

## Purpose

Help the user make a high-stakes or high-friction choice by clarifying the goal, generating or refining options, and ranking them using explicit criteria.

## Use When

- The user is stuck between 2 or more options
- The decision involves uncertainty, tradeoffs, or multiple stakeholders
- The user wants a clear recommendation plus a next-steps plan

## Avoid When

- The user only wants factual info
- The user only wants brainstorming with no decision

## Intake (max 2 questions)

Ask up to 2:

1) What is the decision deadline?

2) What matters most, speed, cost, quality, risk, or learning?

If unanswered, assume a 2 to 4 week horizon and prioritize reversibility, learning, and downside protection.

## Method

1) Restate the decision in one sentence.

2) List options. If the user gave 2 options, add 1 realistic third option.

3) Define decision criteria (5 to 9). Include at least:

- Expected upside
- Downside risk
- Reversibility
- Time and effort
- Alignment with stated values

4) Weight criteria (Low, Medium, High) and explain why.

5) Score each option per criterion (1 to 5) with short evidence notes.

6) Run a sensitivity check. Identify 2 tipping variables that could flip the recommendation.

7) Recommend:

- The best option now
- A fallback option
- A fast experiment or check to reduce uncertainty

## Required Output Structure

## 1) Decision Statement

## 2) Options

## 3) Criteria and Weights

## 4) Scoring Table

## 5) Recommendation

## 6) Risks and Mitigations

## 7) First 7 Days Plan

## 8) Confidence and Tipping Variables
