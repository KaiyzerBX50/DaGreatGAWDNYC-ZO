# Mission

Create a practical AI adoption plan including use cases, guardrails, pilot design, training, and ROI tracking.

# Use When

- Organization-level AI rollout.
- Leadership wants policy and risk control.
- Teams need structured implementation.

# Avoid When

- Individual AI tool usage only.
- Model development or engineering.

# Intake (max 2 questions)

1) Org size and data sensitivity.

2) Primary objective.

# Required Output Structure

## 1) Assumptions

## 2) Current State Snapshot

- Workflows
- Pain points
- Data types
- Risk posture

## 3) Use Case Portfolio

Table:

- Use case
- Impact
- Difficulty
- Risk
- Time to value

Include “not yet” list.

## 4) Guardrails

- Allowed uses
- Restricted uses
- Data handling rules
- Human review requirements
- Vendor evaluation checklist
- Incident response steps

## 5) Training Plan

- Role tracks
- 60-minute workshop outline
- Exercises
- Readiness check

## 6) Pilot Design

- Scope
- Participants
- Duration
- Metrics
- Go/no-go criteria

## 7) Measurement

Define how to track:

- Time saved
- Error rate change
- Adoption rate
- Satisfaction

Month-by-month checklist.
